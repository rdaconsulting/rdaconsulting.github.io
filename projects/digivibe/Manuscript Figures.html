The above figure is a tricky one to implement, because there is overlap in the data – that is, the plot shows 11 distinct lines, but there are actually 20 lines plotted (9 are overlapped by others). The ChatGPT figure does nothing to account for this. One option is to add “transparency” to the lines/dots such that less transparent lines indicate multiple overlapping lines; more transparent lines indicate fewer overlapping lines. Another option is to introduce a bit of ‘jitter’ to the lines so they are barely offset. The latter shows the multiple lines, but it also introduces a bit of inaccuracy because the lines don’t match up perfectly with the pain scores (as you’ll see below, it’s also a bit messier). My suggestion would be to scale the thickness of the line to be greater when there’s overlapping data and thinnest when there are no overlapping trajectories with that exact match.

# dumbell plot
wide.ab <- dv.long |>
  mutate(dg = factor(dg, levels = c("Without Digivibe", "With Digivibe")),
         ptseq = dense_rank(ptid)) |>
  select(ptid, ptseq, dg, abdomen) |>
  pivot_wider(names_from = dg, values_from = abdomen) |>
  rename(pre = `Without Digivibe`, post = `With Digivibe`) |>
  mutate(change = post - pre) |>     # for optional ordering
  arrange(ptseq) |>                  # change to: arrange(desc(change)) to sort by effect
  mutate(x = row_number())           # sequential x-position (necessary if use arrange(desc(change)) to sort by effect; optional otherwise)

wide.ab |> 
  ggplot(aes(x = x)) +
  geom_segment(aes(xend = x, y = pre, yend = post), linewidth = 1) +
  geom_point(aes(y = pre, color = "Without Digivibe"), size = 3) +
  geom_point(aes(y = post, color = "With Digivibe"),  size = 3) +
  scale_color_manual(NULL, values = c("Without Digivibe" = "#E69F00",
                                      "With Digivibe"   = "#56B4E9")) +
  scale_y_continuous("Pain Score (0–10)", limits = c(0, 10), breaks = seq(0, 10, 2)) +
  scale_x_continuous("Patient", breaks = wide.ab$x, labels = wide.ab$ptseq) +
  theme_bw() +
  theme(axis.title = element_text(size = 15),
        axis.text  = element_text(size = 13),
        legend.position = "top")
